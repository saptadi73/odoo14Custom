GRT MRP Overhead Costing

Modul ini membantu memindahkan biaya overhead seperti listrik dan SDM ke cost produksi, lalu merekonsiliasikannya dengan expense aktual agar tidak terjadi double pembiayaan.

Fungsi Utama

  • Simpan overhead bulanan
  • Hitung tarif per kg, per batch, atau per jam
  • Alokasikan overhead ke MO selesai
  • Buat jurnal adjustment

Menu Yang Dipakai

  • Manufacturing > Overhead Periods
  • Manufacturing > Configuration > Overhead Types

Setup Yang Disarankan

  • Listrik Produksi = Per Kg
  • SDM Produksi = Per Kg

Alur Kerja Singkat

  1. Buat atau cek Overhead Type.
  2. Buat Overhead Period bulanan.
  3. Isi actual overhead dari expense/journal atau Manual Actual Amount.
  4. Klik Load Source Entries.
  5. Klik Compute Overhead.
  6. Review basis, rate, absorbed amount, dan variance.
  7. Klik Create Adjustment.
  8. Review dan post jurnal draft jika sudah benar.

Konsep Inti

Tarif overhead = Actual overhead / Total basis alokasi
Applied overhead per MO = Basis MO x Tarif overhead
Variance = Actual overhead - Total applied overhead
Istilah Arti
Actual Overhead Biaya overhead aktual dari expense, jurnal, atau input manual.
Absorbed Overhead Biaya overhead yang dibebankan ke produk berdasarkan basis alokasi.
Variance Selisih antara actual overhead dan absorbed overhead.

Contoh Cepat

Misalnya actual listrik bulan Februari adalah Rp 9.000.000 dan total produksi bulan itu 9.000 kg.

Tarif listrik = 9.000.000 / 9.000 = Rp 1.000 per kg

Kalau satu MO menghasilkan 1.000 kg, maka overhead listrik untuk MO tersebut adalah Rp 1.000.000.

Checklist Bulanan

  • Pastikan semua expense/jurnal overhead sudah masuk.
  • Tag overhead dengan MRP Overhead Type jika memakai source otomatis.
  • Buka periode overhead bulanan.
  • Klik Load Source Entries.
  • Isi Manual Actual Amount jika masih ada biaya yang belum masuk jurnal.
  • Klik Compute Overhead.
  • Review angka sebelum buat jurnal.
  • Klik Create Adjustment.

Troubleshooting Ringkas

Basis Qty = 0

Cek apakah ada MO done di periode itu dan apakah basis alokasi yang dipilih memang cocok dengan data produksi.

Actual Amount = 0

Cek apakah source entries sudah ditarik, expense/jurnal sudah ditag, atau isi manual actual amount bila masih masa transisi.

Variance Besar

Biasanya karena actual overhead belum lengkap, basis alokasi salah, atau tarif manual terlalu tinggi/rendah.
